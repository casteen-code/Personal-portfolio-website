# Casper 个人知识库

这个文件夹是你的职业素材库。它的目标不是直接等于一份简历，而是先把你的经历、项目、能力、数据和证据保存完整，后续再按不同用途提取内容：

- 对外个人总览 HTML 页面
- 针对某个岗位的定制简历
- 面试自我介绍
- 项目复盘和作品集文案

## 你先填什么

建议按这个顺序补充：

1. `00-快速投喂区.md`
   把零散信息先扔进来，不需要整理。
2. `01-个人基础档案.md`
   补齐身份、联系方式、求职方向、公开边界。
3. `02-职业经历素材库.md`
   补每段工作经历里你真正做过什么。
4. `03-项目案例库.md`
   补最能证明能力的项目，尤其是数据、过程和你的贡献。
5. `04-能力证据库.md`
   补不同岗位会用到的能力证据。

## 文件说明

- `00-快速投喂区.md`: 给你随手粘贴素材，后续再整理。
- `01-个人基础档案.md`: 你的稳定基础信息和求职边界。
- `02-职业经历素材库.md`: 按公司和岗位沉淀完整经历。
- `03-项目案例库.md`: 按项目沉淀 STAR 级别的案例。
- `04-能力证据库.md`: 把能力和证据绑定，方便按岗位筛选。
- `05-公开主页内容库.md`: 只放适合公开到个人 HTML 页面的内容。
- `06-岗位定制简历需求单.md`: 每次要做某个岗位简历时复制一份来填。
- `07-简历样式参考.md`: 记录后续岗位定制简历默认沿用的版式规则。

## 使用原则

### 1. 知识库内容可以比简历更长

知识库里要保留完整细节。简历和网页只是从这里挑选，不要为了简历一页纸而提前删掉有价值的信息。

### 2. 事实、结果、证据分开写

尽量补全这几类信息：

- 你负责了什么
- 你怎么做的
- 你和谁协作
- 结果有什么变化
- 数据怎么证明
- 哪些材料可以佐证

### 3. 公开内容和内部素材分开

如果某条信息不能公开，例如公司内部数据、敏感策略、未上线功能、真实后台截图，请写在经历库或项目库，并标注：

`公开限制: 不可公开`

只有确认可以对外展示的内容，才放进 `05-公开主页内容库.md`。

### 4. 针对岗位补证据

同一段经历面对不同岗位，写法会不同。

例如投递产品经理岗位时，需要额外补充：

- 你发现了什么用户问题
- 你如何分析需求优先级
- 你有没有输出需求文档、流程图、埋点方案、版本建议
- 你如何和产品、设计、研发、投放、内容团队协作
- 功能上线后如何验证效果

这些信息如果只出现在“海外营销运营”简历里，通常不够支撑产品经理简历。
