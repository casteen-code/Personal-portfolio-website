# 简历样式参考

## 默认使用方式

后续生成岗位定制简历时，默认参考 `李二虎简历.pdf` 的版式方向，但只沿用格式样式，不复用样例简历中的个人内容、措辞和数据。

## 样式结论

- 默认做一页式中文简历，优先控制在招聘方可快速扫读的密度。
- 页面整体采用单栏正文结构，信息从上到下连续阅读，不做花哨装饰。
- 顶部先放姓名，再放基础信息；头像可放右上角，避免挤占正文宽度。
- 正文按模块分区，模块标题左对齐，并用细横线建立清晰分隔。
- 经历模块采用“左侧公司/岗位，右侧时间”的紧凑行头，下面接项目符号描述。
- 描述条目以结果导向为主，尽量写清动作、方法、指标和业务价值。
- 教育背景、实习/工作经历、其他经历、个人总结、作品集或补充链接可按岗位需要取舍。
- 视觉上以黑白灰为主，强调信息秩序和可读性；少量加粗用于公司、岗位、关键词和关键结果。

## 后续输出参数

这些参数用于我以后生成新简历时落地，不要求和参考 PDF 完全一致：

| 项目 | 默认参数 |
| --- | --- |
| 页面 | A4 纵向 |
| 页数 | 默认 1 页；信息量明显超出时再建议 2 页 |
| 边距 | 上下约 12-16 mm，左右约 14-18 mm |
| 正文字体 | 中文优先用清晰的无衬线字体 |
| 字号 | 姓名约 18-22 pt，模块标题约 11-13 pt，正文约 9-10.5 pt |
| 行距 | 正文紧凑但可读，约 1.1-1.25 |
| 分隔 | 模块标题下使用细横线 |
| 时间 | 经历行头右对齐展示 |
| 条目 | 每段经历优先 2-4 条高价值要点 |
| 链接 | 作品集、主页或项目链接放在末尾补充区 |

## 按岗位定制时的取舍

- 先保留与 JD 最相关的经历和关键词，再压缩次要内容。
- 如果岗位看重投放增长，优先突出预算、渠道、转化、ROAS、素材测试和归因分析。
- 如果岗位看重产品或商业化，优先突出用户问题、漏斗分析、版本协作、功能优化和上线结果。
- 如果岗位看重内容营销，优先突出选题、短剧传播、本地化、KOL/UGC、社群和曝光互动结果。
- 作品链接、工具列表和个人总结要服务岗位匹配度，不为占版面而保留。
